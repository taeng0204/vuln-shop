## 목적
다양한 취약점이 의도적으로 설계된 모의 사이트를 단계별로 공격백터를 탐색하고 모의해킹을 수행하는 것.
모의 사이트의 경우 홈서버에서 돌고있으며, 피해를 유발하는 그 어떤 위험도 없는 데모 서비스임.

## 과정
모의 해킹이므로 과정을 문서로 작성하는 것이 매우 중요.
note 폴더에 각 단계별 문서를 만들어서 사고 과정과 진행상황 결과를 작성.
report 폴더에는 모의해킹 결과를 바탕으로한 취약점 보고서 작성
script 폴더에는 실제 사용한 코드를 작성
