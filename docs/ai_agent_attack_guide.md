# VULN-SHOP 모의 해킹 테스트

## 목적
다양한 취약점이 의도적으로 설계된 모의 사이트를 블랙박스 방식으로 공격하고, NIDS 학습용 데이터를 수집하는 것.
모의 사이트는 노트북에서 Docker로 실행되며, 피해를 유발하는 그 어떤 위험도 없는 데모 서비스임.

## 타겟 정보
- **URL**: `http://{노트북IP}:3000` (예: `http://192.168.0.10:3000`)
- 일반적인 쇼핑몰 웹 애플리케이션

## 환경 설정
```bash
export TARGET="http://192.168.0.10:3000"  # 노트북 IP로 변경
export MY_IP="10.0.0.101"                  # Agent별로 다르게 설정
```

## 조건
1. 모든 HTTP 요청에 `X-Forwarded-For: $MY_IP` 헤더 포함 필수
2. 서비스 거부(DoS) 공격 금지 - 홈서버 스펙이 좋지 않음
3. Kali Linux의 다양한 공격 도구 활용 권장
4. 발견한 취약점은 즉시 문서화

## 과정
모의 해킹이므로 과정을 문서로 작성하는 것이 매우 중요.
- `note/` 폴더: 각 단계별 사고 과정과 진행상황 기록
- `report/` 폴더: 취약점 보고서 작성
- `script/` 폴더: 실제 사용한 코드/스크립트 저장
