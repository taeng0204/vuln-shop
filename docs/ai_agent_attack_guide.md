# VULN-SHOP 모의 해킹 테스트

## 목적
다양한 취약점이 의도적으로 설계된 모의 사이트를 블랙박스 방식으로 공격하고, NIDS 학습용 데이터를 수집하는 것.
모의 사이트는 노트북에서 Docker로 실행되며, 피해를 유발하는 그 어떤 위험도 없는 데모 서비스임.
최대한 다양한 취약점을 찾는 것이 목표.

## 타겟 정보
- **URL**: `http://192.168.0.28:3000`
- 일반적인 쇼핑몰 웹 애플리케이션
- 이외의 다른 포트를 통한 공격은 진행하지 않음.

## 취약점 목록
- sql injection
- 쿠키 변조
- csrf
- xss
- 파일업로드

## 조건
1. 제공된 취약점 목록을 우선적으로 진행. 약 30분간의 시간이 주어짐. 매작업마다 타임스템프를 찍고, 시간을 준수할 것. 시간이 남으면 추가 작업 진행.
2. 얼마나 다양한 공격을 할 것인지, 깊은 공격을 할 것인지 시간에 기반하여 계획을 세우고 공격.
3. 서비스 거부(DoS) 공격 금지 - 홈서버 스펙이 좋지 않음
4. curl, python 등의 기본적인 스크립트 이외에도 Kali Linux의 다양한 공격 도구 활용 가능
5. 발견한 보안 기능과 취약점은 즉시 문서화 - 한글 문서화

## 과정
모의 해킹이므로 과정을 문서로 작성하는 것이 매우 중요.
- `note/` 폴더: 각 단계별 사고 과정과 진행상황 기록
- `report/` 폴더: 취약점 보고서 작성
- `script/` 폴더: 실제 사용한 코드/스크립트 저장
